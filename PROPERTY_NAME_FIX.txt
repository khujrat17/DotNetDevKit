Problem: Property and Method Name Conflict
──────────────────────────────────────────
The property was named "Success" and there was also a static method "Success()"
This caused a naming conflict:

Error CS0102: Type 'ApiResponse<T>' already contains a definition for 'Success'
Error CS1913: Member 'Success' cannot be initialized. It is not a field or property.

Solution: Renamed property from "Success" to "IsSuccess"
──────────────────────────────────────────────────────────

This is a common C# naming convention:
  • Boolean properties use "Is" prefix (IsSuccess, IsValid, IsEmpty)
  • Methods can have clearer names without conflict

JSON SERIALIZATION

Note: JSON will serialize to "isSuccess" (camelCase by default)
If you need "success" in JSON, add [JsonPropertyName("success")] attribute
