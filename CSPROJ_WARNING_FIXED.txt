Warning: NETSDK1080
────────────────────
"A PackageReference to Microsoft.AspNetCore.App is not necessary when 
targeting .NET Core 3.0 or higher. If Microsoft.NET.Sdk.Web is used, 
the shared framework will be referenced automatically. Otherwise, the 
PackageReference should be replaced with a FrameworkReference."

✅ FIXED: Changed PackageReference to FrameworkReference


═══════════════════════════════════════════════════════════════════════════════

WHAT WAS CHANGED
═══════════════════════════════════════════════════════════════════════════════

File: DotNetDevKit.csproj
─────────────────────────

BEFORE:
───────
<ItemGroup>
  <PackageReference Include="Microsoft.AspNetCore.App" Version="6.0.0" />
  <PackageReference Include="Microsoft.Extensions.DependencyInjection" Version="6.0.0" />
  <PackageReference Include="Microsoft.Extensions.DependencyInjection.Abstractions" Version="6.0.0" />
</ItemGroup>

AFTER:
──────
<ItemGroup>
  <FrameworkReference Include="Microsoft.AspNetCore.App" />
</ItemGroup>

Changes Made:
  ✓ Replaced PackageReference with FrameworkReference for Microsoft.AspNetCore.App
  ✓ Removed explicit Version="6.0.0" (automatically resolved)
  ✓ Removed redundant Microsoft.Extensions.DependencyInjection packages
    (automatically included with AspNetCore.App framework reference)


═══════════════════════════════════════════════════════════════════════════════

WHY THIS FIX WORKS
═══════════════════════════════════════════════════════════════════════════════

When targeting .NET 6.0+:
  • The AspNetCore.App framework includes all necessary dependencies
  • Using FrameworkReference is more efficient than PackageReference
  • Avoids version conflicts and simplifies dependency management
  • This is the recommended best practice for .NET 6.0+

The FrameworkReference automatically brings in:
  ✓ Microsoft.AspNetCore.App (entire framework)
  ✓ Microsoft.Extensions.DependencyInjection
  ✓ Microsoft.Extensions.DependencyInjection.Abstractions
  ✓ All other dependencies needed
